# industry-news-tracker

> 行业情报生成器 — 3 阶段 × 4 类 Agent，6 维度并行检索，专为 ODM 老板等决策者打造的细分领域新闻简报。

## 一句话描述

输入**细分领域 + 日期范围**，自动生成一份 6 维度结构化行业情报报告（Markdown + HTML）。

## 适用场景

| 领域类型 | 示例 |
|---------|------|
| 电子制造 | 家用按摩仪、电子烟、储能、机器人、智能门锁、清洁电器 |
| 消费品 | 美妆个护、母婴、家居、日化 |
| 医疗健康 | 家用医疗器械、保健品、医药器械 |
| 工业品 | 仪器仪表、化工原料、零部件 |
| 纯软件 / SaaS | 电子签、协同办公、CRM、HR SaaS、垂直 SaaS |

**不适用的**：宏观金融政策分析、纯学术论文综述。

## 核心架构

```
用户输入（细分领域 + 日期范围）
  ↓
【Phase 1: 6 维度并行检索】
  ├─ Agent 1 → 市场与规模
  ├─ Agent 2 → 前景与趋势
  ├─ Agent 3 → 竞争态势
  ├─ Agent 4 → 客户与渠道
  ├─ Agent 5 → 运营与利润
  └─ Agent 6 → 政策与合规
  ↓
【Phase 2: 综合 Agent】→ 去重 / 同源合并 / 5 要素概述 / 排序 / 生成 Markdown
  ↓
【Phase 3: 边界守卫 + 终止判断】→ 日期检查 / 链接质量 / 维度覆盖 / 缺口补召
  ↓
输出 Markdown → mdstyle 转 HTML
```

## 6 维度定义

| # | 维度 | 关注什么 |
|---|------|---------|
| 1 | 市场与规模 | 市场规模、增长率、区域增量、行业白皮书 |
| 2 | 前景与趋势 | 技术路线演进、产品形态升级、需求变化信号 |
| 3 | 竞争态势 | 新品发布、融资、IPO、产能、专利、人事、战略合作 |
| 4 | 客户与渠道 | 客户画像变化、产品推广、渠道合作、大促数据 |
| 5 | 运营与利润 | 订单变动、供应链、上游元器件报价、上市公司业绩 |
| 6 | 政策与合规 | 出口管制、关税、安全认证、UDI、贸易救济调查 |

## 每条新闻的 5 要素概述范式

| 要素 | 说明 |
|------|------|
| 背景/制度依据 | 依据 X 法 / 政策 / 公告 |
| 硬数据 | 具体数字（规模、增速、价格、覆盖率） |
| 冲击分析 | 对谁意味着什么 |
| 可执行启示 | 备货周期、技术方向、客户取舍 |
| 政策二分（仅政策类） | 区分"打击 X / 合规 Y" |

## 日期范围保障（三道防线）

1. **检索阶段**：关键词带日期修饰词（如 `按摩仪 财报 2026年6月`）
2. **筛选阶段**：看发布日而非事件日（4 月底的财报不纳入 6 月月报）
3. **输出阶段**：边界条目显式标注原始发布日期

## 调用方式

```bash
# 月度报告（默认）
/industry-news-tracker 家用按摩仪 2026.6.1-2026.6.30 --reader ODM老板

# 周报
/industry-news-tracker 智能眼镜 2026.6.24-2026.7.1 --output 周报

# 季报
/industry-news-tracker 储能 2026.Q2 --output 季报
```

或自然语言触发：

> "帮我做一份家用按摩仪 2026 年 6 月的月报，读者是 ODM 老板"
> "做一份智能眼镜的近一周新闻"

## 目录结构

```
industry-news-tracker/
├── SKILL.md                       # 主入口（架构、约束、流程）
├── agents/
│   ├── dim1-market.md             # 维度 1：市场与规模 检索 Agent
│   ├── dim2-trend.md              # 维度 2：前景与趋势 检索 Agent
│   ├── dim3-competition.md        # 维度 3：竞争态势 检索 Agent
│   ├── dim4-channel.md            # 维度 4：客户与渠道 检索 Agent
│   ├── dim5-operation.md          # 维度 5：运营与利润 检索 Agent
│   ├── dim6-policy.md             # 维度 6：政策与合规 检索 Agent
│   ├── agent7-synthesis.md        # 综合 Agent（去重/合并/5要素/输出）
│   ├── agent8-boundary.md         # 边界守卫（日期/链接/覆盖度检查）
│   └── agent9-termination.md      # 终止判断（补召 or 输出）
└── examples/usage.md              # 调用示例与常见错误
```

## 适配新领域

只需修改 6 个 `dimX-*.md` 中的关键词和头部企业清单，主 SKILL.md 不动。

## 配合使用的 Skill

- **mdstyle**：Markdown → HTML（cmd 版式）
- **brainstorming**：生成前对重点关注维度做头脑风暴
- **pdf-to-markdown**：历史报告 PDF 可作为写作参考
