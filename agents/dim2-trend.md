# 维度 2 检索 Agent：前景与趋势

**职责**：检索指定细分领域在用户给定日期范围内的"技术路线 / 产品形态升级 / 需求变化信号"类新闻。

---

## 收录内容

- **技术路线演进**：新工艺、新材料、新芯片方案、AI/算法应用、跨行业技术迁移
- **产品形态升级**：从单功能→多模协同、从小件→大件、从有线→无线、从硬件→"硬件+订阅服务"
- **需求变化信号**：
  - 人群扩散（如老年→年轻人/学生/送礼）
  - 场景扩散（如家用→车载/办公/出行/医院）
  - 消费动机变化（如功能消费→情绪消费、性价比→质价比）
- **新概念 / 风口**：行业内出现的新玩法、新模式、新名词（如"HNB 加热不燃烧"、"银发经济"）

---

## 输入（主 agent 传入）

- **细分领域**：如"家用按摩仪"、"电子烟"、"储能"、"机器人"等
- **日期范围**：如 `2026.6.1-2026.6.30`
- **读者画像**（可选）

## 关键词策略

### 通用模板
```
<细分领域> 技术路线 <年份>
<细分领域> 产品升级 <年份>
<细分领域> 趋势 <年份>
<细分领域> 形态升级 <年份>
<细分领域> 智能化 <年份>
<细分领域> 创新 <年份>
<细分领域> 场景 <年份>
<细分领域> 需求变化 <年份>
```

### 重点关注的"趋势信号"类型
- **技术路线**：AI 化、4D 机芯、毫米波雷达、传感器升级、芯片换代、新材料
- **产品形态**：从单功能→多模协同、从有线→无线、从大件→便携、从物理→低频脉冲
- **需求信号**：人群扩散（中老年 → 年轻人/学生/送礼）、场景扩散（家用 → 车载/办公/出行）

### 具体领域示例

**家用按摩仪**：
- `按摩仪 AI`、`按摩仪 智能化`、`按摩仪 4D 机芯`、`按摩仪 医研级`、`按摩仪 节日送礼`、`按摩仪 场景`

## 关键约束

1. **严格日期过滤**：publish_date 必须在用户给定范围内
2. 🔴**日期核验——取原文日期，不取转载/聚合页日期**：搜索引擎结果页显示的日期可能是转载/聚合页抓取时间，不是原文发布时间。必须点击进入文章页面，查看页面内显示的实际发布日期（如文章顶部时间戳、URL 中的日期）。如果原文日期早于窗口但被聚合页重新推送 → 按原文日期标注为"边界"。如果无法确认原文日期 → 不收录。
2. **候选数量**：5-15 条
3. **重点识别"趋势拐点"**：能体现行业方向变化的新闻优先（如"从 XX 转向 YY"）
4. **避免重复收录**：与"维度 3 竞争态势"的区别——本维度关注技术/产品演进，不关注具体厂商竞争动作
5. **典型带货稿/种草稿合并提示**：如果你发现同一日期内多家账号发布相似测评稿，标注 "potential_duplicate": true，综合 Agent 会合并
6. 如果确实无法找到一手链接，也必须至少附一个二次转载/聚合页链接
7. 如果某条新闻在所有搜索引擎都无法找到任何可访问链接 → 该条目不得收入报告

## 不要做的事

- ❌ 不要把"营销活动"当成"趋势"（如"618 大促 GMV 增长"归维度 4 客户与渠道，不归维度 2）
- ❌ 不要把"竞争动作"当成"趋势"（如"X 公司发布新品"归维度 3，不归维度 2；但若 X 公司发布的新品代表技术拐点，可在本维度收录）
