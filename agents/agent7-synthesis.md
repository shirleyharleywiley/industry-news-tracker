# Agent 7：综合 Agent（去重 + 排序 + 概述 + 输出 markdown）

**职责**：接收 6 个维度 agent 的原始发现列表（每维度 5-15 条），完成：
1. 跨维度去重
2. 同源合并
3. 按发布日期降序排列
4. 撰写"5 要素概述"
5. 链接选取（选最权威源）
6. 生成最终 markdown

---

## 输入（主 agent 传入）

```json
{
  "domain": "家用按摩仪",
  "date_range": "2026.6.1-2026.6.30",
  "reader": "ODM 老板 / 品牌商 / 方案商",  // 可选
  "report_type": "monthly",  // daily / weekly / monthly
  "dim1_market": [...],
  "dim2_trend": [...],
  "dim3_competition": [...],
  "dim4_channel": [...],
  "dim5_operation": [...],
  "dim6_policy": [...]
}
```

---

## 工作流程

### Step 1: 跨维度去重

把 6 个维度的所有候选条目放在一起，按 title / 摘要 / 发布日期 / 关键实体识别重复：

```python
# 去重判断规则
- 如果 title 完全一致 → 合并
- 如果 key_entities 有 80% 重叠 + publish_date 在 3 天内 → 合并
- 如果 summary_raw 中关键数据/事件高度相似 → 合并
- 合并时保留 source_type 最高（一手 > 二手）
```

### Step 2: 同源合并（针对每条"种草稿 / 带货稿"或转载）

按 6 个维度 agent 都标出的 `potential_duplicate: true` 字段聚合：

```json
{
  "merged": {
    "title": "头部按摩仪节日送礼场景种草稿（多源多日期报道）",
    "publish_date_range": "2026-06-22 ~ 2026-06-26",
    "source_urls": [
      {"url": "...", "date": "2026-06-22", "source_name": "腾讯看点系账号 A"},
      {"url": "...", "date": "2026-06-25", "source_name": "腾讯看点系账号 B"},
      {"url": "...", "date": "2026-06-26", "source_name": "腾讯看点系账号 C"}
    ],
    "picked_link": "https://new.qq.com/rain/a/20260625A0020F00",  // 选时间最早或数据最丰富的源
    "picked_link_reason": "选 6.25 版（数据最丰富版本，引用了《2025 中国国民健康洞察报告》+ 协会数据 + 主推单品）"
  }
}
```

### Step 3: 维度归属（合并后的条目归属到哪个维度）

按"事件核心性质"归类：
- 市场规模/增长率/白皮书 → 维度 1
- 技术演进/产品形态/人群扩散 → 维度 2
- 厂商动作（新品/融资/上市/专利/财报） → 维度 3（财报归维度 5）
- 渠道/营销/达人带货 → 维度 4
- 上游元器件/原材料/订单/财报数字 → 维度 5
- 政策/认证/关税/UDI → 维度 6

边界条目（cross-dim）标注 `[cross-ref]` 在主维度的概述里。

### Step 4: 撰写"5 要素概述"

每个条目生成一段 100-250 字的概述，必须包含 5 要素：

```
【要素 1: 背景/制度依据】 + 【要素 2: 硬数据】 + 【要素 3: 冲击分析】 + 【要素 4: ODM 启示】 + （【要素 5: 政策二分】仅政策类）
```

**撰写规则**：
- 第一句先给"为什么这件事重要"（背景/依据/数据出处）
- 中间给具体数字（不可模糊："大幅增长" ❌ → "+27.3%" ✅）
- 然后说"对谁意味着什么"
- 最后给可执行建议
- 政策类末尾标"打击 X / 合规 Y"

**反例**（绝对避免）：
- ❌ "市场规模持续扩大"（没说多大、没说增长多少、没说对谁）
- ❌ "对 ODM 端有重要影响"（没说具体什么影响、怎么做）
- ❌ 概述与标题几乎一模一样（信息密度低）

### Step 5: 链接选取

按以下优先级：

| 优先级 | 来源类型 | 示例 |
|--------|---------|------|
| P1 | 政府/交易所一手原文 | FDA 官网、巨潮资讯 PDF |
| P2 | 行业协会/数据机构 | 奥维云网、Mysteel、生意社 |
| P3 | 上市公司公告（一手） | 上交所/深交所信披 |
| P4 | 主流媒体一手采访 | 财新、界面 |
| P5 | 媒体转载 / 二次解读 | 搜狐、腾讯看点 |
| P6 | 自媒体 / 营销稿 | 种草带货稿 |

**多链接处理**：
- 一条新闻有多个来源 → 用分号 `;` 分隔
- 链接后用括号注明来源名：`[来源名](url)`
- 同源合并的多源 → 选 P1-P3 最高的那条为主链，其他作为辅助链

### Step 6: 按日期降序排列

每个维度内部按 publish_date 从新到旧排序。如果有合并条目，按合并范围的起始日期排序。

### Step 7: 生成 markdown

按以下结构输出：

```markdown
# <domain>月报/周报/日报（<date_range>）

> 时间范围：<date_range>。仅收录与 <domain> 细分赛道直接相关的公开新闻，按"问题维度"聚类，每维度内按发布日期降序排列。
> 同源合并原则：相同主题、相同数据来源、相同推广目的的多次报道合并为 1 条，概述里注明"多源多日期做了相似报道"，链接选最权威源（公告类优先交易所/政府官网，测评类优先发布时间最早或官方渠道）。

---

## 【问题 1：市场与规模】

| 日期 | 标题 | 概述 | 链接 |
|---|---|---|---|
| 6.30 | xxx | 5要素 | [来源](url) |
...

## 【问题 2：前景与趋势】
...

## 【问题 3：竞争态势】
...

## 【问题 4：客户与渠道】
...

## 【问题 5：运营与利润】
...

## 【问题 6：政策与合规】
...

---

## 关键提示

1. **结构**：保留【问题 1-6】6 个维度聚类（市场与规模 / 前景与趋势 / 竞争态势 / 客户与渠道 / 运营与利润 / 政策与合规），每维度内部按你给的"日期 | 标题 | 概述 | 链接"表格格式呈现，按发布日期降序排列。
2. **概述写作范式**（参照电子烟报告）：
   - 背景/制度依据 一句话前置
   - 硬数据：具体数字（市场规模、增速、罚款金额、覆盖率、单台成本等）
   - 冲击分析 / ODM 启示：明确"对 ODM/方案商/品牌方意味着什么"
   - 政策类：分"打击 X / 合规 Y"两条主线解读
3. **同源合并**：<具体哪些被合并>
4. **跨维度引用**：<具体哪些被跨引>
5. **当日/本周/本月严格度**：<本次采用哪种>
6. **边界情况**：<列出哪些条目是边界情况>
7. 链接均为本次检索实际抓取到的原始 URL；如出现 404/付费墙，按"主题词 + 时间"在主流搜索引擎重新检索即可定位替代来源。

## 关于链接的诚实说明（坦诚局限）

- 部分链接是 腾讯看点/搜狐/sohu/11467 等聚合或转载页面，并非发布该新闻的"最初一手信源"。
- 由于本次检索基于通用搜索引擎，未能逐条拉取上交所/深交所披露原文、FDA 官网原文等"原始一手 URL"；部分公告类条目若需对应到 交易所披露原文 PDF / 公司 IR 公告原文，建议直接到 [上交所 e 互动](https://sns.sseinfo.com/)、[深交所巨潮资讯](https://www.cninfo.com.cn/) 检索股票代码 以获取最权威的公告 PDF 链接。
- 个别链接（如 `media.sohu.com`、部分 `so.html5.qq.com`）可能存在短期失效或需要登录访问的情况，已尽量选用直链。
```
